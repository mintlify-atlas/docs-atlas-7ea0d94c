## Component overview

The Todo List application consists of two main components:

- **App**: Main application component managing state and rendering the todo list
- **FormDialog**: Reusable dialog for adding and editing todos

## App component

The App component is the root component that manages all application state and renders the todo list interface.

### Import and dependencies

```tsx App.tsx
import { useEffect, useState } from "react";
import "./App.css";
import { Button, Card, CardContent, IconButton, TextField, Typography, Box } from "@mui/material";
import FormDialog from "./FormDialog";
import type { Todo } from "./types/todo";
```

### Key methods

#### handleTodoSubmit

Handles both creating new todos and updating existing ones:

```tsx App.tsx
const handleTodoSubmit = (todo: Todo) => {
  setTodoList((prevTodoList) => {
    const next = new Set(prevTodoList);
    if (editingTodo) next.delete(editingTodo);
    
    next.add({...todo, completed: editingTodo?.completed ?? false});
    return next;
  });
  setIsDialogOpen(false);
  setEditingTodo(null);
};
```

<Note>
When editing, the function preserves the completion status from the original todo.
</Note>

#### toggleComplete

Toggles a todo's completion status:

```tsx App.tsx
const toggleComplete = (todo: Todo) => {
  setTodoList((prevTodoList) => {
    const next = new Set(prevTodoList);
    next.delete(todo);
    next.add({ ...todo, completed: !todo.completed });
    return next;
  });
};
```

#### deleteTodo

Removes a todo from the list:

```tsx App.tsx
const deleteTodo = (todo: Todo) => {
  setTodoList((prevTodoList) => {
    const next = new Set(prevTodoList);
    next.delete(todo);
    return next;
  });
};
```

#### handleEdit

Prepares a todo for editing:

```tsx App.tsx
const handleEdit = (todo: Todo) => {
  setEditingTodo(todo);
  setIsDialogOpen(true);
};
```

### Render structure

The component renders:

1. **Search field**: TextField for filtering todos by name
2. **Add Todo button**: Opens dialog for creating new todos
3. **FormDialog**: Modal dialog for adding/editing todos
4. **Clear button**: Removes all todos
5. **Todo cards**: List of filtered todos with glassmorphic styling

## FormDialog component

The FormDialog is a reusable component for creating and editing todos with built-in validation.

### Props interface

```tsx FormDialog.tsx
type FormDialogProps = {
  onSubmit: (todo: Todo) => void;
  open: boolean;
  onClose: () => void;
  initialTodo?: Todo | null;
};
```

<ParamField path="onSubmit" type="(todo: Todo) => void" required>
  Callback function called when form is submitted with validated todo
</ParamField>

<ParamField path="open" type="boolean" required>
  Controls dialog visibility
</ParamField>

<ParamField path="onClose" type="() => void" required>
  Callback function called when dialog should close
</ParamField>

<ParamField path="initialTodo" type="Todo | null">
  Optional todo to pre-fill form for editing. If null/undefined, form is empty for creating new todo.
</ParamField>

### Import and dependencies

```tsx FormDialog.tsx
import type { FormEvent } from "react";
import Button from "@mui/material/Button";
import TextField from "@mui/material/TextField";
import Dialog from "@mui/material/Dialog";
import DialogActions from "@mui/material/DialogActions";
import DialogContent from "@mui/material/DialogContent";
import DialogContentText from "@mui/material/DialogContentText";
import DialogTitle from "@mui/material/DialogTitle";
import { MenuItem, Select } from "@mui/material";
import { subjects, todoSchema, type Todo } from "./types/todo";
```

### Form submission handler

The component uses Zod for runtime validation:

```tsx FormDialog.tsx
const handleSubmit = (event: FormEvent<HTMLFormElement>) => {
  event.preventDefault();
  const formData = new FormData(event.currentTarget);
  const formJson = Object.fromEntries(formData.entries());
  const todo = todoSchema.parse(formJson);
  onSubmit(todo);
  onClose();
};
```

<Note>
`todoSchema.parse()` will throw an error if validation fails. Material-UI's built-in HTML5 validation prevents most invalid submissions.
</Note>

### Usage example

From the App component:

```tsx App.tsx

  onSubmit={handleTodoSubmit}

```

## Component best practices

1. **Type safety**: Both components use TypeScript interfaces for props
2. **Controlled/uncontrolled inputs**: Dialog uses `defaultValue` for uncontrolled form inputs
3. **Accessibility**: Proper `aria-label` attributes on IconButtons
4. **Separation of concerns**: Form logic isolated in FormDialog
5. **Validation**: Client-side HTML5 validation + Zod runtime validation
