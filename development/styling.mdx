## Overview

The Todo List application uses a combination of Material-UI's `sx` prop, Emotion (Material-UI's styling engine), and custom CSS files to create a glassmorphic user interface with a dark theme.

## Styling approach

The application employs three styling methods:

1. **Material-UI sx prop**: Inline component styling with theme integration
2. **Global CSS**: Base styles and CSS custom properties in `index.css`
3. **Component CSS**: Component-specific styles in `App.css`

## Glassmorphic design

The application's signature visual style is the glassmorphic effect applied to todo cards.

### Glassmorphic card styling

Implemented in `src/App.tsx:100-111`:

```tsx App.tsx
<Card
  sx={{
    margin: "8px 16px",
    background: "linear-gradient(135deg, rgba(255,255,255,0.16), rgba(255,255,255,0.02))",
    borderRadius: 3,
    border: "1px solid rgba(255, 255, 255, 0.4)",
    boxShadow: "0 16px 40px rgba(15, 23, 42, 0.7)",
    backdropFilter: "blur(18px)",
    WebkitBackdropFilter: "blur(18px)",
  }}
>
```

### Glassmorphic effect breakdown

<Tabs>
  <Tab title="Background gradient">
    ```css
    background: linear-gradient(135deg, rgba(255,255,255,0.16), rgba(255,255,255,0.02));
    ```
    
    Creates a subtle diagonal gradient from semi-transparent white (16% opacity) to nearly transparent (2% opacity).
  </Tab>
  <Tab title="Border and shadow">
    ```css
    border: 1px solid rgba(255, 255, 255, 0.4);
    boxShadow: 0 16px 40px rgba(15, 23, 42, 0.7);
    borderRadius: 3;
    ```
    
    - Semi-transparent white border (40% opacity) for glass edge effect
    - Large, soft shadow for depth
    - Border radius for rounded corners (theme spacing unit * 3 = 24px)
  </Tab>
  <Tab title="Backdrop blur">
    ```css
    backdropFilter: blur(18px);
    WebkitBackdropFilter: blur(18px);
    ```
    
    Applies 18px blur to content behind the card, creating the frosted glass effect. `WebkitBackdropFilter` ensures Safari compatibility.
  </Tab>
</Tabs>

<Info>
The glassmorphic effect requires both `backdropFilter` and `WebkitBackdropFilter` for cross-browser support.
</Info>

## Material-UI sx prop

The application extensively uses Material-UI's `sx` prop for component styling.

### TextField styling

```tsx App.tsx
<TextField
  sx={{
    margin: "8px 16px",
    input: { color: "white" },
    label: { color: "white" }
  }}
  id="todo-search"
  label="Search by name"
  variant="standard"
  value={search}
  onChange={(e) => setSearch(e.target.value)}
  fullWidth
/>
```

<Note>
The `sx` prop allows nested selectors to style child elements like `input` and `label`.
</Note>

### Typography styling

Todo title with conditional styling based on completion status:

```tsx App.tsx
<Typography
  variant="h6"
  sx={{
    color: "inherit",
    textDecoration: todo.completed ? "line-through" : "none",
    opacity: todo.completed ? 0.7 : 1,
  }}
>
  {todo.name}
</Typography>
```

Completed todos are styled with:
- Line-through text decoration
- Reduced opacity (70%)

### Metadata typography

```tsx App.tsx
<Typography sx={{ color: "rgba(255,255,255,0.85)" }}>
  {todo.subject} • Priority {todo.priority}
</Typography>
<Typography sx={{ color: "rgba(255,255,255,0.7)" }}>
  {todo.date.toLocaleDateString()}
</Typography>
```

Uses varying opacity levels to create visual hierarchy:
- Primary metadata: 85% opacity
- Secondary metadata (date): 70% opacity

## Theme integration

Material-UI components inherit from the default theme:

- `borderRadius: 3` = theme spacing unit × 3 = 24px
- Consistent spacing with `margin: "8px 16px"`
- Standard Material-UI variants (`outlined`, `standard`)

## Emotion styling engine

Material-UI v7 uses Emotion for CSS-in-JS:

```json package.json
{
  "dependencies": {
    "@emotion/react": "^11.14.0",
    "@emotion/styled": "^11.14.1",
    "@mui/material": "^7.3.5"
  }
}
```

Benefits:
- Type-safe styling with TypeScript
- Theme-aware styling
- Dynamic styles based on props
- Automatic critical CSS extraction

## Best practices

1. **Use sx prop for component styles**: Keeps styles co-located with components
2. **Leverage theme values**: Use theme spacing and breakpoints for consistency
3. **RGBA for transparency**: Enables layering and glassmorphic effects
4. **Semantic naming**: Use descriptive `aria-label` attributes
5. **Conditional styling**: Use ternary operators in `sx` for dynamic styles
6. **Cross-browser compatibility**: Include vendor prefixes (`WebkitBackdropFilter`)
