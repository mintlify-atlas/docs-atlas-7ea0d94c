Get your Todo List application running locally in just a few steps.

## Prerequisites

Before you begin, ensure you have the following installed:

- **Node.js** 18 or higher
- **pnpm**, **npm**, or **yarn** package manager

<Note>
This project uses pnpm by default, but you can use npm or yarn if you prefer.
</Note>

## Installation

<Steps>
  <Step title="Clone the repository">
    Clone the project to your local machine:
    
    ```bash
    git clone https://github.com/moderj/todo-list.git
    cd todo-list
    ```
  </Step>

  <Step title="Install dependencies">
    Install all required dependencies using your preferred package manager:
    
    <CodeGroup>
    ```bash pnpm
    pnpm install
    ```
    ```bash npm
    npm install
    ```
    ```bash yarn
    yarn install
    ```
    </CodeGroup>
    
    <Info>
    The project includes React 19, TypeScript, Vite, Material-UI, and Zod validation.
    </Info>
  </Step>

  <Step title="Start the development server">
    Run the development server:
    
    <CodeGroup>
    ```bash pnpm
    pnpm dev
    ```
    ```bash npm
    npm run dev
    ```
    ```bash yarn
    yarn dev
    ```
    </CodeGroup>
    
    The application will start at `http://localhost:5173` (default Vite port).
  </Step>
</Steps>

## Creating your first todo

Once the application is running:

<Steps>
  <Step title="Click Add Todo button">
    Click the **Add Todo** button at the top of the page to open the todo creation dialog.
  </Step>

  <Step title="Fill in the todo details">
    Enter the following information in the form:
    
    - **Name**: A descriptive name for your todo (e.g., "Complete project documentation")
    - **Subject**: Choose either "avtash" or "hagnash" from the dropdown
    - **Priority**: Enter a number from 1-10 (1 = lowest, 10 = highest)
    - **Date**: Select a due date using the date picker
    
    <Info>
    All fields are validated using Zod schemas. Priority must be between 1 and 10, and all fields are required.
    </Info>
  </Step>

  <Step title="Submit the form">
    Click **Submit** to create your todo. It will appear in the list with:
    
    - The todo name as the heading
    - Subject and priority displayed below
    - The selected date
    - Action buttons for completing, deleting, and editing
  </Step>
</Steps>

## Interactive features

### Mark todos as complete

Click the checkmark (✓) icon on any todo card to toggle its completion status. Completed todos display with:
- Strike-through text
- Reduced opacity
- Green checkmark icon

### Edit existing todos

Click the edit (✏️) icon to open the form dialog pre-filled with the todo's current data. Make your changes and submit to update.

### Delete todos

Click the delete (🗑️) icon to remove a todo from the list immediately.

### Search todos

Use the search field at the top to filter todos by name. The search:
- Filters in real-time with a 300ms debounce
- Is case-insensitive
- Matches any part of the todo name

### Clear all todos

Click the **Clear** button to remove all todos from the list at once.

## Building for production

When you're ready to build the application for production:

<CodeGroup>
```bash pnpm
pnpm build
```
```bash npm
npm run build
```
```bash yarn
yarn build
```
</CodeGroup>

This runs TypeScript compilation followed by Vite's production build. The output will be in the `dist/` directory.

### Preview the production build

To preview the production build locally:

<CodeGroup>
```bash pnpm
pnpm preview
```
```bash npm
npm run preview
```
```bash yarn
yarn preview
```
</CodeGroup>

## Available scripts

The project includes these npm scripts from `package.json`:

- `dev` - Start the Vite development server with hot module replacement
- `build` - Compile TypeScript and build for production
- `lint` - Run ESLint on the codebase
- `preview` - Preview the production build locally

## Troubleshooting

<Accordion title="Port 5173 is already in use">
  Vite will automatically try the next available port. Check the terminal output to see which port is actually being used, or modify `vite.config.ts` to specify a different port.
</Accordion>

<Accordion title="Type errors during build">
  Ensure your TypeScript version matches the project requirement (~5.9.3). Run `pnpm install` to ensure all dependencies are correctly installed.
</Accordion>

<Accordion title="Material-UI styles not loading">
  The project uses Emotion as the CSS-in-JS solution. Ensure both `@emotion/react` and `@emotion/styled` are installed correctly.
</Accordion>
