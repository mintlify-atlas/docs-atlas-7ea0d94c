The todo list application provides two key organizational features: priority levels and subjects. These help you categorize and rank your todos based on importance and context.

## Priority levels

Priority levels allow you to rank your todos from 1 to 10, with higher numbers indicating higher priority.

### Understanding priority levels

The priority system works as follows:

- **Range**: 1 to 10 (inclusive)
- **Scale**: Higher numbers = higher priority
- **Required**: Every todo must have a priority level
- **Validation**: The application enforces the 1-10 range using Zod schema validation

<Info>
  Priority levels are validated on both the frontend and through the Zod schema. If you enter a number outside the 1-10 range, it will be coerced to the nearest valid value.
</Info>

### Setting priority levels

When creating or editing a todo:

<Steps>
  <Step title="Locate the priority field">
    In the todo form dialog, find the **"priority"** field. It's a number input field that appears after the subject dropdown.
  </Step>

  <Step title="Enter a priority value">
    Type a number between 1 and 10:
    
    - **1-3**: Low priority tasks
    - **4-6**: Medium priority tasks
    - **7-9**: High priority tasks
    - **10**: Critical/urgent tasks
    
    <Tip>
      Develop your own priority system that works for you. Some users reserve 10 for truly urgent tasks, while others use a more distributed scale.
    </Tip>
  </Step>

  <Step title="Priority is saved with your todo">
    The priority level is stored as part of the todo and displayed in the todo card.
  </Step>
</Steps>

### Viewing priority levels

Each todo card displays its priority level in the bottom section:

- **Location**: Bottom left of the card
- **Format**: "subject • Priority X" (e.g., "avtash • Priority 8")
- **Always visible**: The priority is shown for all todos, both complete and incomplete

<Note>
  While priority levels are set and displayed, the application doesn't automatically sort todos by priority. Todos appear in the order they were added to the list.
</Note>

### Priority best practices

<Accordion title="Be consistent with your priority scale">
  Establish a personal system for what each priority level means:
  
  - Use 1-3 for "nice to have" tasks
  - Use 4-6 for regular work items
  - Use 7-9 for important deadlines
  - Reserve 10 for true emergencies
  
  Consistency helps you quickly understand task importance at a glance.
</Accordion>

<Accordion title="Don't overuse high priorities">
  If everything is priority 10, nothing is priority 10. Be selective:
  
  - Limit the number of priority 9-10 tasks
  - Use the full range of the scale
  - Reassess priorities regularly
  
  This ensures your high-priority items truly stand out.
</Accordion>

<Accordion title="Update priorities as situations change">
  Use the edit feature to adjust priority levels when:
  
  - Deadlines become more urgent
  - Blockers are removed
  - Requirements change
  - Tasks become less relevant
  
  Keeping priorities current makes them more useful for decision-making.
</Accordion>

## Subjects

Subjects provide categorical organization for your todos. They help you group related tasks together.

### Available subjects

The application includes two predefined subject categories:

1. **avtash**
2. **hagnash**

<Info>
  These subjects are defined in the codebase as `subjects = ["avtash", "hagnash"]` and are enforced through Zod enum validation. Custom subjects are not currently supported without modifying the source code.
</Info>

### Selecting a subject

When creating or editing a todo:

<Steps>
  <Step title="Open the subject dropdown">
    In the todo form dialog, locate the **subject** dropdown menu. It appears after the name field.
  </Step>

  <Step title="Choose from available subjects">
    Click the dropdown to see the list of subjects:
    
    - avtash
    - hagnash
    
    Select the subject that best categorizes your todo.
  </Step>

  <Step title="Subject is required">
    You must select a subject before submitting the form. The form validation ensures a subject is always chosen.
  </Step>
</Steps>

### How subjects appear

Subjects are displayed in the todo card:

- **Location**: Bottom left, before the priority level
- **Format**: "subject • Priority X"
- **Example**: "avtash • Priority 5"

<Note>
  The subject and priority are displayed together as a single line, separated by a bullet point (•) for clear visual separation.
</Note>

### Subject organization strategies

<Accordion title="Use subjects for context switching">
  If the subjects represent different contexts or projects:
  
  - Assign all related tasks to the same subject
  - Use subjects to mentally organize your work
  - Focus on one subject at a time when working through todos
  
  This helps maintain focus and reduces context switching.
</Accordion>

<Accordion title="Combine subjects with priority">
  Create a system that uses both:
  
  - High-priority avtash tasks for urgent work in that category
  - Low-priority hagnash tasks for future planning
  - Balance your work across subjects based on priority
  
  This two-dimensional organization provides flexibility in task management.
</Accordion>

<Accordion title="Establish subject definitions">
  Define what each subject means for your workflow:
  
  - Document the purpose of each subject
  - Create guidelines for which tasks belong to which subject
  - Review subject assignments periodically
  
  Clear definitions prevent confusion when categorizing new todos.
</Accordion>

## Combining priorities and subjects

The most effective todo management uses both features together.

### Organizational matrix

You can think of your todos in a matrix:

```
                avtash          hagnash
High (7-10)    Urgent tasks    Critical items
Medium (4-6)   Regular work    Standard tasks
Low (1-3)      Future ideas    Backlog items
```

### Workflow example

<Steps>
  <Step title="Create a new todo">
    You need to prepare a presentation for next week:
    
    - **Name**: "Prepare Q1 presentation"
    - **Subject**: avtash
    - **Priority**: 7 (high, but not urgent)
    - **Date**: Next week's date
  </Step>

  <Step title="Adjust priority as deadline approaches">
    As the deadline gets closer, edit the todo:
    
    - Keep **Subject**: avtash
    - Update **Priority**: 9 (now urgent)
    - Everything else stays the same
  </Step>

  <Step title="Mark complete when done">
    After finishing the presentation:
    
    - Click the checkmark to complete the todo
    - The todo remains visible with its subject and priority
    - It has strikethrough styling to show completion
  </Step>
</Steps>

<Tip>
  Review your todos regularly and update priorities as deadlines approach or circumstances change. The combination of subject and priority creates a flexible system that adapts to your changing needs.
</Tip>

## Validation and data integrity

The application uses Zod schema validation to ensure data quality:

### Priority validation

```typescript
priority: z.coerce.number().min(1).max(10)
```

- Coerces input to a number type
- Enforces minimum value of 1
- Enforces maximum value of 10
- Prevents invalid priority values

### Subject validation

```typescript
subject: z.enum(["avtash", "hagnash"])
```

- Only allows the two predefined subjects
- Prevents typos or invalid values
- Ensures consistent categorization

<Warning>
  The validation happens when you submit the form. If you enter invalid values (like priority 15 or a custom subject), the form submission will fail with a validation error.
</Warning>

## Customizing subjects

If you need different subject categories, you'll need to modify the source code:

1. Edit `/src/types/todo.ts`
2. Update the `subjects` array with your categories
3. The Zod enum validation automatically uses the new subjects

<Note>
  This requires developer access to the codebase. The application doesn't provide a user interface for customizing subjects.
</Note>
